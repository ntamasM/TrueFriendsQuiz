## True Friends Quiz — v2.2.0

### Producer QA feedback (round 2)

This release responds to the AirConsole producer's second round of QA feedback:
audio, language consistency, and a snappier question-picking flow.

**Audio**

- Fixed background music not playing — track URLs used a root-absolute path
  (`/Assets/...`) that 404'd in the deployed build (served from a sub-path). Now
  resolved via `import.meta.env.BASE_URL`, matching every other asset.
- Music now starts on the lobby/waiting screen (on `onReady`) instead of only
  after the master presses "Start game", and keeps playing through the session.
- Added a best-effort autoplay-unlock fallback for browsers that block
  `audio.play()` without a prior user gesture.

**Language consistency**

- Every player's phone now follows the language selected on the main screen, for
  both the UI text and the questions. Previously each phone used its own device
  language, which produced a mixed-language experience (screen-language questions
  with device-language UI). New `set_language` message; the controller drops the
  per-device browser auto-detect and adopts the language broadcast by the screen
  (on change, on connect, and on every gameplay message).

**Question picking — auto-assign + reroll**

- Replaced the 4-question chooser with a single auto-assigned question plus a
  **Reroll** button (up to 5 rerolls per round).
- The offered question and its answers are mirrored to the TV with a "Scream for
  the question you want!" prompt, so the whole group reacts to each roll.
- Category voting is unchanged and remains host-toggleable in settings.

**Polish**

- Pick-view buttons now fill their column (removed dead space beside them).
- Removed the redundant category label from the TV question preview.
- Moved the "scream" prompt below the rerolls counter on the TV.

**Docs & metadata**

- Bumped version to 2.2.0 (package.json, README, About page).
- Updated Home and About pages with the new features.
